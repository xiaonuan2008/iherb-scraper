# iHerb 爬虫使用教程（Web 版之前）

> 适用阶段：命令行脚本版本。Web 界面做好后会另说，但底层逻辑不变。
> 已实测可用：collagen 品类，8/8 商品完整拿到成分含量 + UPC。

---

## 0. 环境准备（首次必做）

需要 **Python 3.10+** 和 **Playwright 浏览器**：

```bash
# 1. 安装 Python 依赖
pip install playwright beautifulsoup4

# 2. 安装 Chromium 浏览器（爬虫靠它过 Cloudflare）
playwright install chromium
```

> 没装浏览器会报 `Executable doesn't exist` 之类的错，装完就好。

下载脚本后，在脚本目录里运行：

```bash
cd iherb_scraper
python3 run.py
```

---

## 1. 文件结构（你只需要动 1 个文件）

| 文件 | 作用 | 你要不要改 |
|------|------|-----------|
| **`config.py`** | 所有爬取配置（关键词/品牌/限速/输出） | ✅ **主要改这里** |
| `run.py` | 运行入口，直接 `python3 run.py` | ❌ 不用动 |
| `iherb_scraper.py` | 浏览器封装 + 页面解析 + 导出 | ⚠️ 仅「加新字段」时改 |
| `output/` | 跑出来的 CSV 结果 | ❌ 自动生成，别手改 |

**一句话：换品类改 `config.py` 一行，加字段才碰 `iherb_scraper.py`。**

---

## 2. 爬不同的产品：改 `config.py` 哪里

打开 `config.py`，从上到下改这几项：

### 场景 A：换品类 / 原料关键词

```python
# 单个品类：
KEYWORDS = ['collagen']

# 多个品类批量跑（每个关键词独立翻页，结果合并去重）：
KEYWORDS = ['collagen', 'vitamin c', 'omega-3', 'probiotics', 'magnesium']
```

> 关键词就是 iHerb 搜索框里输的东西。中文站支持中英文，比如 `'胶原蛋白'` 也行。

### 场景 B：按品牌爬

**方式 1（推荐）：关键词 + 品牌过滤**
先搜品类，再只保留目标品牌：

```python
KEYWORDS = ['collagen']
BRAND_FILTER = ['NOW Foods', 'California Gold Nutrition', 'Sports Research']
```

**方式 2：只爬某个品牌的全线产品**
把品牌名当关键词搜（iHerb 搜索支持品牌名）：

```python
KEYWORDS = ['NOW Foods']
BRAND_FILTER = []   # 留空即可
```

> `BRAND_FILTER` 是模糊匹配——填 `'NOW'` 就能匹配 `NOW Foods` 全系。

### 场景 C：控制爬多少（防封 / 省时间）

```python
MAX_PAGES    = 2     # 每个关键词最多翻几页（每页约 48 个商品）
DETAIL_LIMIT = 0     # 0 = 爬全部详情；N = 只爬前 N 个（快速验证用）
```

> 首次跑建议 `DETAIL_LIMIT = 5` 先验证，确认数据 OK 再设回 `0` 跑全量。

### 场景 D：中英文站切换

```python
BASE_URL = 'https://cn.iherb.com'   # 中文站
# BASE_URL = 'https://www.iherb.com' # 英文站
LOCALE   = 'zh-CN'                   # 跟着 BASE_URL 改：英文站用 'en-US'
```

## 3. 怎么运行

```bash
cd iherb_scraper
python3 run.py
```

跑完看 `output/` 目录：

| 文件 | 内容 | 用途 |
|------|------|------|
| `iherb_products.csv` | 每个商品一行（名称/品牌/价格/规格/UPC/成分摘要…） | 总表，直接 Excel 打开 |
| `iherb_ingredients.csv` | 每个成分一行（商品ID/成分名/含量/每日值%） | 成分级分析、去重、对比 |

两个文件通过 `product_id` 关联。

---

## 4. 输出字段说明

**`iherb_products.csv` 列：**
`product_id` / `part_number` / `brand` / `brand_id` / `title` / `price` / `currency` / `gtin12`(UPC) / `sku` / `category` / `weight_value` / `weight_unit` / `rating` / `review_count` / `serving_size` / `servings_per_container` / `has_amount` / `needs_ocr` / `other_ingredients` / `allergens` / `warnings` / `sales_text` / `is_out_of_stock` / `url` / `image_url`

**`iherb_ingredients.csv` 列：**
`product_id` / `brand` / `title` / `ingredient_name` / `amount` / `daily_value` / `serving_size`

> `has_amount=False` 且 `needs_ocr=True` 的商品：文字成分表无含量，需下载 `label_images` 里的标签图做 OCR（目前 iHerb 实测几乎都有文字含量，此情况极少）。

---

## 5. 加新字段（进阶，改 `iherb_scraper.py`）

当 iHerb 改版、或你想多抓一个信息（比如「保质期」「发货地」），改 `iherb_scraper.py`：

1. 在 `_parse_detail()` 里加一段解析，把值写进 `product` 字典：
   ```python
   product['新字段名'] = 解析结果
   ```
2. 在文件底部的 `product_cols` 列表里加上 `'新字段名'`，否则不会写进 CSV。

> 搜索页字段在 `_parse_search()`，同理处理。

---

## 6. 注意事项（重要）

| 问题 | 说明 | 应对 |
|------|------|------|
| **Cloudflare 验证** | iHerb 有反爬，纯 requests 会被 403 | 已用 Playwright 浏览器自动过，**无需登录/验证码** |
| **连续请求限流** | 同一浏览器连续访问会触发中文挑战页且不放行 | 已处理：每次请求开**新浏览器 context**（新 session 首请求必过） |
| **爬太快被拦** | 频率过高可能升级验证 | 保持 `DETAIL_DELAY ≥ 2`、`PAGE_DELAY ≥ 3`；若大面积失败就调大 |
| **分页参数** | 搜索翻页用 `&p=2` | 已知可用，第 2 页起若返回 0 个就是没更多结果 |
| **`needs_ocr` 商品** | 极少数无文字含量 | 下载 `label_images` 用 PaddleOCR 补（后续单独脚本） |

## 7. 常见命令速查

```bash
# 快速验证（只爬前 5 个详情）
# 先把 config.py 里 DETAIL_LIMIT 改成 5，再：
python3 run.py

# 全量跑（DETAIL_LIMIT 设回 0）
python3 run.py

# 换品类：改 config.py 的 KEYWORDS，再 python3 run.py
```
